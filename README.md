# 海斗战报 MVP

网页端海克斯大乱斗赛后复盘原型，支持已登录 LOL 客户端读取、同区玩家检索、演示数据、CSV/JSON 导入、六职业评分、历史回城死亡规则、白天模式和深夜模式。

## 在线版本

[打开海斗战报在线版](https://haxiaxiaozio-art.github.io/lol-haidou/?v=7)

在线网站可以直接体验演示数据和 CSV/JSON 导入。读取真实 LOL 战绩时，在同一台 Windows 电脑上安装[海斗数据助手](https://github.com/haxiaxiaozio-art/lol-haidou/releases/latest/download/HaiDouHelperSetup.exe)，并在浏览器提示时允许访问本地网络。普通玩家不再需要安装 Node.js，也不需要下载源码。

V17 安装器不再写死 C 盘目录。安装者必须选择一个当前用户可写的位置，实际程序放在所选位置的 `HaiDouHelper` 子目录；安装仍只写当前用户的 HKCU 注册项，不申请管理员权限，也拒绝 Windows、Program Files、ProgramData 和磁盘根目录。网页提示未连接时，可以点击“已安装，启动助手”。卸载入口位于 Windows 设置的“已安装的应用”，卸载前会二次校验安装位置与产品标记，不删除所选的父目录。

当前公开安装包尚未使用受信任的商业代码签名，因此 Windows 首次运行仍可能显示“未知发布者”。构建脚本已支持通过 `HAIDOU_SIGN_CERT_SHA1` 和 Windows SDK `signtool.exe` 对助手与安装器进行 SHA-256 时间戳签名；正式消除未知发布者需要购买可信 OV/EV 代码签名证书，或改用 Microsoft Store 的 MSIX 分发。自签名证书不适合公开下载，也不会自动获得 SmartScreen 信誉。

## 运行

普通玩家直接下载安装包即可。以下源码运行方式仅面向开发者，需要 Node.js 22.13 或更高版本。

最简单的方式：先启动 LOL 并登录，再双击项目根目录里的 `启动网站.cmd`。启动脚本会检查并开启仅限本机访问的数据助手，然后直接打开正式网站，不再占用 3000 系列开发端口。网页顶部会自动显示“数据助手未启动”“等待 LOL 登录”或当前 Riot ID。

也可以在终端运行：

```powershell
npm install
npm run dev
```

旧的 `启动数据助手.cmd` 仍可用于开发调试。正式使用建议安装助手，网页会每隔数秒自动重新检测，不需要反复刷新。

## 读取 LOL 客户端

1. 启动并登录国服 LOL 客户端，停留在客户端主页即可。
2. 打开海斗战报，选择“我的战绩”。
3. 点击“检测登录客户端”，确认 Riot ID 和大区。
4. 选择读取 20、40、100 或 200 场，点击“筛选海斗并生成评分”。

检索其他玩家时，选择“检索玩家”，输入完整的 `游戏名#尾标` 并选择场次。当前版本通过已登录客户端的本机会话查询同大区玩家，暂不支持未登录检索或跨大区检索。

本地数据助手只监听 `127.0.0.1:3212`，只接受本机网页请求，不读取 LOL 或 WeGame 密码。操作评分、英雄、海克斯、出装和逐局详情仍只在本机处理；网络估算分仅向线上服务提交经过 SHA-256 匿名化的玩家关系、胜负、对局时间与版本，不上传 Riot ID、昵称或完整战绩。它读取 League Client 本地接口；该接口不是 Riot 正式支持的稳定公共 API，客户端更新后可能需要同步调整连接器。

## 网络估算分

V16 数据助手优先通过本机 SGP 读取战绩，SGP 不可用或返回不足时自动回退到 LCU，再通过本机日志补读旧局。SGP、LCU 和日志会先转换成统一比赛模型，再进入筛选与评分。同步诊断中心会区分客户端未登录、大区不可用、接口超时或异常、连接权限不足和战绩字段缺失。SGP 访问令牌只在数据助手进程内存中使用，不会返回网页，也不会上传到网络服务。助手仍会把匿名胜负关系提交到独立的 Cloudflare Worker 与 D1 数据库，生成带区间、置信度和校准状态的稳定趋势分。玩家还可以主动勾选贡献去标识化的本人海斗指标，用于六职业真实样本校准；该开关默认关闭，检索其他玩家不会贡献样本。公网接口部署在 `lol-haidou-rating.haxiaxiaozio.workers.dev`，生产库不会保存 Riot ID、原始 PUUID 或客户端令牌。

## 真实样本校准与数据质量

社区校准模型分别统计六个职业。单个职业达到 100 份有效样本后，才会开始调整该职业的四项指标基线；副职业奖励、高光阈值和死亡惩罚使用独立门槛。每份校准样本必须对应网络估算服务已接收的匿名比赛，同一比赛最多保留 10 位玩家的样本。

V16 增加校准安全治理。异常样本会根据字段冲突、极端死亡率、职业指标与历史分布偏离等规则进入隔离表，不参与模型计算。新模型先保存为版本快照，经过参数漂移护栏后只向 10% 的匿名稳定分桶灰度发布；候选质量不合格或近 7 天异常隔离率超过阈值时自动回滚。管理接口支持调整 1% 到 50% 的灰度比例、提前晋级和手动回滚，所有状态均保留生效版本、候选版本、父版本、质量分、策略版本和回滚落点。

V17 完成剩余 P0。真实样本、异常窗口、候选版本和回滚链按游戏补丁分别治理，避免不同版本装备与数值改动互相污染。`/api/calibration/replay` 支持按模型版本，或按补丁与历史时间点恢复评分模型；网页导出的 JSON 报告会内嵌当时的模型快照，再次导入时按原版本重放，不会被未来模型静默改分。线上服务对读取、评分提交、样本提交和治理接口使用分级匿名限流，只保存分钟级客户端哈希；`/api/health` 提供一小时服务摘要，每日任务自动清理过期限流桶、30 天健康事件、365 天匿名比赛与校准样本，以及 730 天前的废弃模型版本。连续错误可通过 `ALERT_WEBHOOK_URL` 发送去重告警。

数据质量面板展示扫描返回、海斗筛选、职业识别、核心战斗字段、装备覆盖和回城时序精度。质量分只解释数据是否完整，不参与玩家操作评分；网络模型不可用时，页面会明确回退到 `rules-2026.07` 固定基线。

正式构建使用：

```powershell
npm run build
```

## 数据导入

打开页面中的“导入自己的 CSV / JSON”，下载模板后替换演示字段。文件只在浏览器页面中读取，不会上传。

CSV 的 `role` 字段仅接受：辅助、法师、刺客、坦克、射手、战士。`augments` 使用竖线分隔。历史回城局可填写 `recall_pick_minute`、`deaths_before_recall` 和 `deaths_after_recall`。
